# Приложение для тренера — бриф

Приложение для **одного тренера**, который ведёт своих подопечных. Заменяет связку
«Календарь айфона + Заметки + счёт занятий в голове».

Платформа: **нативное iOS (SwiftUI + SwiftData)**. Данные хранятся локально на телефоне.
Один пользователь — тренер. Клиенты в приложение не заходят.

---

## Принцип №1 — «просто как Заметки»

Это главное требование. Приложение НЕ должно быть похоже на сложную CRM.

- Любое частое действие — в 1–2 тапа (отметить занятие проведённым, продлить пакет).
- Минимум обязательных полей. Новый клиент = достаточно одного имени.
- Никаких настроек, мастеров, многошаговых форм на старте.
- Главный экран отвечает на два вопроса тренера: **«кого тренирую сегодня?»** и
  **«кому пора платить?»**

---

## Главная ценность

Тренер всегда видит по каждому клиенту: **оплачено → проведено → осталось**, и понимает,
**когда пора брать деньги снова**. Это та боль, ради которой делается приложение.

---

## Модель данных

### Клиент
- Имя (обязательно)
- Телефон / телеграм (необязательно)
- Цель / заметки — свободный текст (как в Заметках)
- Статус: **активный / пробный / новый / архив**
- Список пакетов
- Список занятий

### Пакет (гибкий, не только «10»)
- Тип: **пакет на N занятий** (5, 10, любое число) / **разовое** / **пробное** / **абонемент на период**
- Количество занятий (для пакетов) или срок (для абонемента)
- Цена
- Дата оплаты
- Остаток считается автоматически: куплено − проведено

> Важно: тренер берёт по-разному. Дизайн должен спокойно показывать и «10 занятий»,
> и «пробное 1 занятие», и «разовое». Не завязываться на цифру 10.

### Занятие
- Клиент
- Дата и время
- Статус: **запланировано / проведено / отменено**
- Привязано к пакету (из какого пакета списывается)
- При статусе «проведено» — списывается 1 занятие из текущего пакета

---

## Экраны (для дизайнера)

### 1. Сегодня (главный)
- Список занятий на сегодня по времени.
- У каждого: имя клиента, время, остаток по пакету (напр. `осталось 3 из 10`).
- Один тап по занятию → «Проведено» (списывает занятие). В идеале свайп.
- Сверху можно листать на другие дни (вчера / завтра).

### 2. Клиенты
- Список всех клиентов.
- У каждого крупно виден **остаток занятий**.
- Цветовой сигнал:
  - 🟢 занятий много
  - 🟡 осталось 1–2 → **скоро надо продлевать**
  - 🔴 пакет закончился / 0 → **пора брать оплату**
- Пробные/новые клиенты помечены отдельно.
- Поиск по имени.

### 3. Карточка клиента
- Контакты (тап по телефону/телеграму → позвонить/написать).
- Цель и заметки.
- Текущий пакет: оплачено / проведено / осталось.
- Кнопка **«Продлить / новый пакет»**.
- История занятий (даты, статусы).
- История оплат (когда и сколько платил) — для учёта денег.

### 4. Добавить занятие
- Выбрать клиента, дату, время.
- Сохранение → событие автоматически попадает в Календарь айфона.

### Экран денег (опционально, можно позже)
- Сводка: сколько получено за месяц, кто сколько должен / у кого заканчивается пакет.

---

## Интеграция с Календарём айфона

- Используем `EventKit` (родной механизм Apple).
- **Источник правды — приложение.** Создаёшь занятие в приложении → оно пишет событие
  в Календарь (в отдельный календарь «Тренировки», чтобы не мешалось с личным).
- Направление **одностороннее** на старте: приложение → Календарь
  (создать / изменить / удалить событие). Двусторонняя синхронизация (читать чужие
  правки из Календаря обратно) сложнее — откладываем.
- Тренер по-прежнему видит расписание в привычном Календаре айфона, но ведёт его из
  приложения, где оно связано с клиентом и оплатой.

---

## Что НЕ делаем в первой версии (чтобы не усложнять)

- Аккаунты для клиентов / клиентское приложение.
- Сервер и облако (всё локально; iCloud-бэкап можно включить позже одной галочкой).
- Программы тренировок / база упражнений (пока остаются в Заметках; добавим во вторую версию).
- Онлайн-оплаты внутри приложения (учёт денег — да, приём платежей — нет).
- Двусторонняя синхронизация с Календарём.

---

## Этапы

1. **MVP:** Клиенты + гибкие пакеты + остаток занятий + расписание + запись событий в Календарь.
2. **V2:** Экран денег (доход за месяц), программы тренировок, iCloud-бэкап.
3. **V3 (если нужно):** клиентский доступ, push-напоминания.
